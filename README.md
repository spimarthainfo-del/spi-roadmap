# SPI 個別学習ロードマップ

SPI対策の生徒一人ひとりに、**「いつまでに・どの優先順で・どのチャネル（YouTube/アプリ/確認テスト）で・何をやるか」** を表示する個別Web画面です。
生徒ごとに固有URLを発行してLINE等で共有できます。

- ビルド不要・依存ゼロ（HTML+CSS+JavaScriptのみ）
- データは `data.js` を書き換えるだけ。プログラミング知識は不要
- 進捗チェックはブラウザに自動保存

---

## 1. 使ってみる（ローカル）

`index.html` をダブルクリックしてブラウザで開くだけです。

- 生徒一覧が出る → 生徒を選ぶ
- もしくは直接：`index.html?id=tanaka-7x9k2m`（田中さん）/ `index.html?id=sato-3p8w5n`（佐藤さん）

各単元は **①▶️動画 → ②📱アプリ → ③📝確認テスト** の3ステップ。
チェックを入れると完了になり、自動保存されます。

---

## 2. 生徒を追加する（かんたん：入力画面）

`input.html` を開く（一覧の「＋ 新規作成」からでもOK）。

- 生徒名・本番日・テスト形式・1日の学習量を入力
- **「まとめて貼り付け」** に、手元のメモをそのまま貼れます。例：

  ```
  ・集合
  ・損益
  ———6月24まで
  ・割合
  ・表
  ————6月30まで
  ・推論（ここがよく出ます）
  ————7月12まで
  ```

  「この内容で読み込む」を押すと、単元グループと締切（「6月24まで」「7月末まで」も解釈）に自動変換されます。
  「損益→損益算」「速度→速度算」などの略称や「（メモ）」付きでもそのまま使え、教材リンクも効きます。
- 各行は下のエディタで微調整可。「ロードマップを作成して開く」で即表示されます。

> 入力画面で作った生徒は**このブラウザに保存**されます（端末ごと）。全端末で共有・本番運用するときは、次の `data.js` に書く方法を使ってください。

## 2b. 生徒を追加する（本番運用：data.js を直接編集）

`data.js` の `STUDENTS` に1ブロック追加するだけです。

```js
{
  id: "yamada-k4m9z2",          // 推測されにくい文字列にする（URLになる）
  name: "山田 次郎",
  examDate: "2026-08-20",        // テスト本番日
  examType: "ペーパーテスト",     // テストセンター / Webテスティング / ペーパーテスト
  startDate: "2026-06-18",       // 学習開始日
  dailyLoad: "毎日45分",          // 表示用の目安
  plan: [                        // 優先順に上から。due＝中間締切
    { topics:["割合","損益算"], due:"2026-06-30" },
    { topics:["集合","表の読み取り"], due:"2026-07-10" },
    // …本番日まで続ける
  ],
  // seedDone は任意（最初から完了扱いにしたいステップ）
},
```

- `topics` に書く単元名は、`TOPICS`（単元マスタ）のキーと**完全一致**させること。
- テスト形式ごとのおすすめ優先順位は `data.js` 冒頭のコメントに記載。コピペして `due` を埋めてください。
- 個別URLは `index.html?id=（設定したid）`。これを生徒に共有します。

---

## 3. 教材リンクを差し替える

`data.js` の `TOPICS` で、各単元の以下を実URLに変更：

- `youtube` … コツ動画
- `app` … トレーニングアプリ（リンク無しなら `#` のまま＝「開く」ボタンは非表示）
- `test` … 確認テスト（Google Drive等。`#` なら「開く」非表示）

---

## 4. 生徒に共有する

### ステップ1：画面本体を一度だけネットに置く（無料）

フォルダごと置くだけで動きます。**生徒データはURLに載るので、置くのは画面本体（HTML/CSS/JS）だけ**でOK。

- **Netlify Drop** … <https://app.netlify.com/drop> にこの `spi_schedule` フォルダをドラッグ&ドロップ → 即 `https://○○.netlify.app/` が発行される
- **Vercel** … `vercel` コマンドでデプロイ
- **GitHub Pages** … リポジトリに置いて Pages を有効化

### ステップ2：生徒ごとの共有リンクを送る

公開後、ロードマップ画面の **「📤 このロードマップのリンクを生徒に共有」** を押すとリンクがコピーされます
（入力画面の「作成済み」一覧の **「共有リンク」** ボタンからでもOK）。

リンクは `https://○○.netlify.app/index.html#d=……` の形で、**生徒の情報がリンク自体に入っています**。
これをLINE等で送れば、生徒はその端末で自分のロードマップを開けます（データの保存・サーバー不要）。

> **共有リンクの注意点**
> - 生徒データはリンクに焼き込まれます。**あとで予定を変えたら、リンクを作り直して送り直す**必要があります。
> - 生徒がつけたチェック（進捗）は**その生徒の端末内だけ**に保存され、講師側には戻りません。
> - 「講師が生徒の進捗を見たい」「編集を自動反映したい」場合は、サーバー連携（Firebase / スプレッドシート等）への拡張が必要です。

---

## 5. 仕組み・制約メモ

- **スケジュール生成**：各単元を3ステップに分解し、`startDate` 〜 各 `due` の期間に均等配置。「今日やること」「今週」「単元別」の3粒度で表示。最終締切が本番日を過ぎると警告が出ます。
- **進捗の保存場所**：見ている端末・ブラウザの `localStorage`。生徒が自分のスマホで開けば自分の進捗が貯まります（運用者の端末とは別管理）。
- **次の拡張候補**：複数端末で進捗共有（要サーバー/スプレッドシート連携）、本番日からの締切自動逆算、達成バッジ・通知など。
